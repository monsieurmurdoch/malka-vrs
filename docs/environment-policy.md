# Environment and Base-URL Policy

The app must never guess its way from local or staging into production. Every
surface needs an explicit environment and tenant source.

## Environment Names

Use only these runtime names:

- `local`: developer machine, local Docker Compose, disposable data.
- `staging`: production-like rehearsal with isolated database, staging domains, Stripe test keys, and Twilio sandbox/test credentials.
- `production`: live customer traffic.

Set `NODE_ENV` to `development`, `staging`, or `production` as appropriate. For
mobile/build-time tenant choice, set `TENANT`, `VRS_TENANT`, or
`EXPO_PUBLIC_TENANT` to one of:

- `malka`
- `malkavri`
- `maple`
- `malka-staging`
- `malkavri-staging`
- `maple-staging`

## Web

Web builds get tenant config from `whitelabel-runtime.js`, generated by:

```sh
TENANT=malka node scripts/whitelabel-prebuild.js
TENANT=maple-staging node scripts/whitelabel-prebuild.js
```

Policy:

- Local web may use same-origin relative API/WS URLs.
- Staging web must be built with a `*-staging` tenant config.
- Production web must be built with the production tenant config.
- A staging host must not serve production-generated `whitelabel-runtime.js`.

## Native Mobile

Native domain resolution comes from static tenant config, cached tenant config,
or build environment. Staging internal builds must set the staging tenant ID at
build/install time. Production app-store builds must set the production tenant ID.

Policy:

- No staging mobile build may rely on the default `malka` fallback.
- No staging mobile build may ship with `vrs.malkacomm.com`,
  `vri.malkacomm.com`, or `vri.maplecomm.ca` as its API/queue target.
- Every mobile smoke note must record app name, bundle/application ID, tenant ID,
  API base, queue WebSocket base, and device used.

## API and Queue

| Surface | Local | Staging | Production |
| --- | --- | --- | --- |
| Malka VRS API/queue | `localhost:3001` | `staging-vrs.malkacomm.com` | `vrs.malkacomm.com` |
| Malka VRI API/queue | `localhost:3001` | `staging-vri.malkacomm.com` | `vri.malkacomm.com` |
| Maple VRI API/queue | `localhost:3001` or local Maple service | `staging-vri.maplecomm.ca` | `vri.maplecomm.ca` |

The queue WebSocket URL is derived from the same tenant domain:

- HTTPS host -> `wss://<host>/ws`
- HTTP local host -> `ws://<host>/ws`

## Ops/Admin

Ops/admin endpoints use `/ops` behind nginx:

- Local: `http://localhost:3003` or `http://localhost:8080/ops`
- Staging: `https://staging-admin.malkacomm.com/ops` or the tenant host `/ops`
- Production: `https://admin.malkacomm.com/ops` or the tenant host `/ops`

Admin tests must include tenant filters so Maple and Malka activity cannot be
mistaken for each other.

## Twilio

Local and staging must use Twilio test credentials, a sandbox project, or no
Twilio credentials at all. Production Twilio credentials are forbidden in
`.env.staging`.

Base URLs:

- Local: `http://localhost:3002`
- Staging: `https://staging-vrs.malkacomm.com/twilio`
- Production: `https://vrs.malkacomm.com/twilio`

## Stripe

Use:

- `BILLING_STRIPE_MODE=mock` for local development without Stripe.
- `BILLING_STRIPE_MODE=test` with `sk_test_*` and `whsec_*` for staging.
- `BILLING_STRIPE_MODE=live` with `sk_live_*` only in production after finance sign-off.

Staging invoice/customer IDs must never be copied into production records.

Invoice automation:

- `BILLING_AUTO_INVOICE_ENABLED=true` enables the daily invoice auto-run inside
  the VRS server. Keep unset or `false` in local/dev unless intentionally
  rehearsing billing.
- `BILLING_AUTO_INVOICE_SEND=false` generates/updates due invoices without
  sending them. Omit or set `true` only when Stripe/email delivery is ready for
  that environment.
- `BILLING_AUTO_INVOICE_HOUR_UTC=6` controls the once-daily UTC run hour.
- `BILLING_INVOICE_CC_EMAILS` is a comma-separated business-copy list for
  finance/admin visibility. Do not put personal test addresses in production.

## Verification

Before promoting any build, record:

- Tenant ID and environment.
- API, queue, ops, and Twilio base URLs.
- Database name and host class, not password.
- Stripe mode.
- Seed account set.
- Smoke command output.
